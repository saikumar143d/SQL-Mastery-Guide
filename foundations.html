Database Foundations

This section breaks down the core concepts of relational databases, from the ground up.

SQL (Structured Query Language) is the standard language for managing relational databases. To build a solid foundation, you should master the following core concepts, starting from the basic building blocks and moving toward more complex operations.

The Four Sub-Languages of SQL

SQL commands are categorized into four main groups. Understanding this division helps clarify the purpose of each command.

DDL (Data Definition Language)

Defines and manages the database structure.

  • CREATE: Make a new database, table, or view.
  • ALTER: Modify an existing table's structure.
  • DROP: Permanently delete a database or table.
  • TRUNCATE: Quickly remove all rows from a table.

DML (Data Manipulation Language)

Used to interact with the data within tables.

  • SELECT: Retrieve data from one or more tables.
  • INSERT: Add new rows of data into a table.
  • UPDATE: Modify existing rows in a table.
  • DELETE: Remove specific rows from a table.

DCL (Data Control Language)

Manages user permissions and access rights.

  • GRANT: Give specific permissions to a database user.
  • REVOKE: Take away permissions from a user.

TCL (Transaction Control Language)

Manages transactions (sequences of operations).

  • COMMIT: Save all changes made in a transaction.
  • ROLLBACK: Undo all changes made in a transaction.
  • SAVEPOINT: Set a point within a transaction to roll back to.

The SELECT Statement: The Heart of SQL

The SELECT statement is the most important and widely used command for retrieving data. A complete query follows a specific logical order of operations.

  • SELECT: Specifies the columns you want to retrieve. Use * to select all columns.
  • FROM: Specifies the table you are querying from.
  • WHERE: Filters rows based on specific conditions.
  • GROUP BY: Aggregates rows into summary rows.
  • HAVING: Filters groups created by the GROUP BY clause.
  • ORDER BY: Sorts the final result set.
  • LIMIT / TOP: Restricts the number of rows returned.

Example Query Structure:

SELECT column1, column2
FROM table_name
WHERE condition
GROUP BY column1
HAVING condition_on_group
ORDER BY column2 DESC
LIMIT 10;

JOINs: Combining Data

JOIN clauses are used to combine rows from two or more tables based on a related column between them. The diagrams below illustrate how each join type works.

INNER JOIN

Returns only the records that have matching values in both tables.

LEFT JOIN

Returns all records from the left table (A), and the matched records from the right table (B).

RIGHT JOIN

Returns all records from the right table (B), and the matched records from the left table (A).

FULL OUTER JOIN

Returns all records when there is a match in either the left (A) or right (B) table.

Constraints & Data Types

These rules ensure the integrity and reliability of the data in your database.

Common Data Types

  • Text: VARCHAR, TEXT, CHAR
  • Numeric: INT, FLOAT, DECIMAL
  • Date/Time: DATE, TIME, TIMESTAMP
  • Boolean: BOOLEAN

Constraints

  • PRIMARY KEY: Uniquely identifies each record.
  • FOREIGN KEY: Links two tables together.
  • UNIQUE: Ensures all values in a column are different.
  • NOT NULL: Ensures a column cannot have a NULL value.
  • DEFAULT: Provides a default value for a column.
  • CHECK: Ensures values satisfy a specific condition.

Important Functions & Operators

Aggregate Functions

Perform a calculation on a set of values and return a single value.

  • COUNT(): Counts the number of rows.
  • SUM(): Calculates the sum of values.
  • AVG(): Calculates the average of values.
  • MIN() / MAX(): Returns the smallest/largest value.

Set Operators

Combine the result sets of two or more SELECT statements.

  • UNION: Combines two result sets and removes duplicate rows.
  • UNION ALL: Combines two result sets but includes all duplicate rows.
  • INTERSECT: Returns only the rows that appear in both result sets.
  • EXCEPT: Returns rows from the first query that are not present in the second.

Advanced Foundational Concepts

Once you are comfortable with the above, these concepts are the next step to a solid foundation.

  • Subqueries (Nested Queries): A query placed inside another query to perform complex filtering and data retrieval.
  • Common Table Expressions (CTEs): A temporary, named result set (using WITH) that improves readability of complex queries.
  • Views: A virtual table based on an SQL statement, used to simplify queries and provide security.
  • Indexes: Special lookup tables that the database can use to speed up data retrieval significantly.
  • Database Normalization: The process of organizing tables to minimize data redundancy (e.g., 1NF, 2NF, 3NF).

SQL vs. NoSQL: A Deeper Look

The core difference is structure. Relational databases organize data into rigid, predefined tables like spreadsheets, while non-relational databases use flexible models to store varied data types, like files in a folder. SQL is the universal language used to communicate specifically with relational databases.

Relational Databases (SQL)

Relational databases store data in a highly structured way, using tables composed of rows and columns.

  • Schema: The structure is defined by a fixed schema before you add any data.
  • Relationships: Data in different tables is linked together using keys.
  • Consistency: Designed for reliability and consistency (ACID compliance).

Non-Relational Databases (NoSQL)

Non-relational databases are built for flexibility and scale, storing unstructured or semi-structured data.

  • Schema: They have a dynamic schema, meaning you can add data without a predefined structure.
  • Data Models: Several types exist, including Document, Key-Value, and Graph.
  • Scalability: Designed to scale out horizontally (adding more servers).
